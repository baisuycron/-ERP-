# 新零售 ERP 商品资料管理

基于 `Vite + React` 搭建的商品资料管理前端示例项目，用于演示新零售 ERP 中商品主数据模块的基础能力。

当前版本聚焦以下核心场景：

- 商品资料查询
- 商品新建
- 商品编辑
- 商品查看
- 商品启用 / 停用
- 商品逻辑删除
- 商品导入
- 商品导出
- SKU 与条码维护

## 技术栈

- React 18
- Vite 5
- 原生 CSS
- LocalStorage 本地持久化

## 启动方式

先安装依赖：

```bash
npm install
```

启动开发环境：

```bash
npm run dev
```

默认开发地址：

```text
http://127.0.0.1:5173
```

## 构建命令

执行生产构建：

```bash
npm run build
```

构建产物默认输出到：

```text
dist/
```

本地预览构建结果：

```bash
npm run preview
```

## 页面说明

当前页面为“商品资料管理”单页，主要包含以下区域：

### 1. 顶部操作区

- 下载导入模板
- 导入商品
- 新建商品

### 2. 统计卡片区

用于展示当前商品主数据概况：

- 商品总数
- 停用商品数
- SKU 总数
- 有效条码数

### 3. 查询筛选区

支持按以下条件筛选：

- 商品编码
- 商品名称
- 品牌
- 商品状态
- 一级分类
- 二级分类
- 三级分类

### 4. 列表区

列表展示字段包括：

- 商品编码
- 商品名称
- 品牌
- 三级分类
- 主单位
- SKU 数
- 条码数
- 状态
- 更新时间

支持操作：

- 查看
- 编辑
- 启用 / 停用
- 删除
- 导出当前结果
- 导出勾选项

### 5. 商品编辑抽屉

可维护以下信息：

- 基础信息
- 分类信息
- 单位信息
- 价格信息
- SKU 信息
- 条码信息

## 当前数据说明

- 页面数据默认存储在浏览器 `LocalStorage`
- 首次打开会自动生成一批示例数据
- 后续新建、编辑、删除、启停用操作会直接更新本地数据

这意味着当前版本更适合：

- 原型演示
- 前端交互评审
- 后端接口联调前的页面预演
