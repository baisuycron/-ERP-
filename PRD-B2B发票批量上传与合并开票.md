## 2. 需求背景

B2B 交易场景下，买家在平台下单后，订单会流转至卖家店铺。买家基于订单发起开票申请，卖家在线下完成发票开具后，需要将发票 PDF 文件与订单映射关系批量上传至平台。

当前业务需要支持合并开票场景：一张发票可以关联多笔订单。平台需要在导入时完成文件校验、发票校验、订单校验和合并开票一致性校验，并在校验通过后更新订单开票状态。

## 3. 需求目标

### 3.1 业务目标

- 支持卖家通过 ZIP 包批量上传发票 PDF 和订单映射关系。
- 支持一张发票关联多笔订单的合并开票场景。
- 保证发票与订单关系准确、唯一、可追溯。
- 支持导入失败明细导出，便于卖家修正后重新上传。

### 3.2 产品目标

- 上传流程清晰，卖家知道如何准备文件。
- 错误反馈明确，失败原因可定位到发票级或订单级。
- 批次导入结果可查看、可导出、可追踪。
- 功能作为“发票管理”的下级页面呈现，保持后台导航一致性。

## 4. 业务范围

### 4.1 支持范围

- 卖家通过 ZIP 包批量上传发票。
- ZIP 内包含 `pdf/` 文件夹和 `mapping.xlsx` 映射表。
- PDF 文件必须按 `发票号码.pdf` 命名。
- 系统根据 Excel 中的发票号码自动匹配同名 PDF。
- 支持一张发票对应多笔订单。
- 支持失败明细导出。

### 4.2 不支持范围

- 一个订单拆分到多张发票。
- 已上传成功的发票修改关联订单。
- 无 Excel 映射关系时，仅凭 PDF 自动识别订单关系。
- PDF 文件名不按发票号码命名时自动纠错。

## 7. 业务流程

### 7.1 主流程

1. 买家基于订单发起开票申请。
2. 卖家线下开具发票 PDF。
3. 卖家整理 ZIP 包，包含 PDF 文件夹和 Excel 映射表。
4. 卖家进入发票管理，点击“批量上传发票”。
5. 卖家下载模板并填写发票信息、订单关联信息。
6. 卖家上传 ZIP 包并点击“导入”。
7. 系统解析 ZIP 和 Excel。
8. 系统按发票维度执行校验。
9. 校验通过：生成发票记录、生成订单关联、更新订单为已开票。
10. 校验失败：整张发票导入失败，记录失败原因。
11. 卖家查看导入结果并导出失败明细。

### 7.2 异常流程

- ZIP 无法解析：导入失败，提示文件格式异常。
- 缺少 `mapping.xlsx`：导入失败，提示缺少映射表。
- 缺少同名 PDF：对应发票失败。
- 发票号码重复：对应发票失败。
- 任一关联订单失败：整张发票失败。

## 8. 上传文件设计

### 8.1 ZIP 结构

```text
invoice_upload.zip
├─ pdf/
│  ├─ FP20260001.pdf
│  ├─ FP20260002.pdf
├─ mapping.xlsx
```

### 8.2 PDF 命名规则

- PDF 文件必须按 `发票号码.pdf` 命名。
- 系统根据 Excel 中的发票号码自动匹配同名 PDF。
- 文件名仅用于上传匹配，不作为唯一业务主键。
- 发票唯一性仍按发票号码判重。

### 8.3 Excel 结构

建议使用两个 Sheet。

#### Sheet1：发票信息

一行表示一张发票。

| 字段 | 是否必填 | 说明 |
| --- | --- | --- |
| 发票号码 | 是 | 用于判重，并匹配同名 PDF |
| 发票类型 | 是 | 如电子普通发票、电子增值税专用发票 |
| 开票日期 | 是 | 日期格式合法 |
| 发票抬头 | 是 | 买方开票抬头 |
| 纳税人识别号 | 是 | 买方税号 |
| 发票金额 | 是 | 必须大于 0 |

#### Sheet2：订单关联

一行表示一张发票关联一笔订单。

| 字段 | 是否必填 | 说明 |
| --- | --- | --- |
| 发票号码 | 是 | 必须存在于 Sheet1 |
| 订单号 | 是 | 需要关联的订单 |

## 10. 核心业务规则

### 10.1 基础规则

- 一个订单只能对应一张发票，不允许拆票。
- 发票唯一性按发票号码判重。
- 当前仅有订单维度开票申请，无开票申请单实体。
- PDF 文件需按 `发票号码.pdf` 命名。

### 10.2 合并开票规则

同一张发票关联的订单必须同时满足：

- 同一发票抬头 / 纳税人识别号。
- 同一发票类型。
- 同一店铺订单。

### 10.3 导入处理规则

- 系统按发票维度处理导入。
- 一张发票关联多笔订单时，不允许部分失败。
- 任一订单校验失败，则该张发票整体导入失败。

整票失败时：

- 不生成发票记录。
- 不生成订单关联记录。
- 不更新任何订单开票状态。

## 11. 校验规则

### 11.1 ZIP 级校验

| 校验项 | 失败提示 |
| --- | --- |
| ZIP 是否可正常解压 | ZIP 文件解析失败 |
| 是否存在 `mapping.xlsx` | 未找到 mapping.xlsx |
| 是否存在 PDF 文件 | 未找到 PDF 文件 |
| 文件格式是否正确 | 文件格式错误 |
| Excel 模板结构是否正确 | Excel 模板结构错误 |

### 11.2 发票基础校验

| 校验项 | 失败提示 |
| --- | --- |
| 发票号码不能为空 | 发票号码不能为空 |
| 发票号码不能重复 | 发票号码已存在，禁止重复上传 |
| ZIP 中必须存在同名 PDF | 未找到与发票号码对应的 PDF 文件 |
| 发票类型不能为空 | 发票信息填写不完整 |
| 开票日期不能为空且格式合法 | 开票日期格式错误 |
| 发票抬头不能为空 | 发票信息填写不完整 |
| 纳税人识别号不能为空 | 发票信息填写不完整 |
| 发票金额不能为空且大于 0 | 发票金额必须大于 0 |

### 11.3 映射关系校验

| 校验项 | 失败提示 |
| --- | --- |
| Sheet2 发票号码必须存在于 Sheet1 | 订单关联中的发票号码不存在 |
| 同一发票号码只能对应一套发票信息 | 同一发票号码存在多套发票信息 |
| 同一张发票至少关联 1 笔订单 | 发票未关联订单 |
| 同一发票号码只能匹配 1 个同名 PDF | PDF 文件重复 |

### 11.4 订单业务校验

| 校验项 | 失败提示 |
| --- | --- |
| 订单存在 | 订单不存在 |
| 订单属于当前店铺 | 订单不属于当前店铺 |
| 订单状态允许开票 | 订单当前状态不允许开票 |
| 订单未开票 | 订单已开票，禁止重复关联 |
| 订单未被其他发票占用 | 订单已被其他发票占用 |

### 11.5 合并开票一致性校验

| 校验项 | 失败提示 |
| --- | --- |
| 发票抬头 / 纳税人识别号一致 | 同一张发票关联订单的发票抬头 / 纳税人识别号不一致 |
| 发票类型一致 | 同一张发票关联订单的发票类型不一致 |
| 店铺一致 | 同一张发票关联订单不属于同一店铺 |

## 16. 风险与注意事项

- 当前按发票号码判重，后续可能存在跨主体号码冲突风险。
- 一张发票关联订单数暂不限制，后端建议增加保护阈值。
- PDF 文件按发票号码命名，需要约束发票号码字符范围，避免文件系统不兼容字符。
- 页面展示关联订单过多时，后续可增加分页或搜索。
- 导入失败不应产生半成功数据，避免发票与订单关系不一致。

## 17. 验收标准

### 17.1 页面验收

- 点击发票管理列表页的“批量上传发票”，顶部新增 `批量上传发票` 页签。
- `批量上传发票` 页签为 `发票管理` 下级页面。
- 关闭 `批量上传发票` 页签后返回 `发票管理`。
- 页面展示三步导入引导、上传表单、注意事项、导入按钮。

### 17.2 导入成功验收

- 上传合法 ZIP 后可成功解析。
- 一张发票关联多笔订单时，全部通过则整票成功。
- 成功导入后订单状态更新为已开票。
- 导入结果展示成功发票数。

### 17.3 导入失败验收

- 任一关联订单失败时，整张发票导入失败。
- 失败时不落发票记录、不落关联、不更新订单状态。
- 导入结果展示失败发票数。
- 可导出失败明细。

### 17.4 文件规则验收

- PDF 文件必须按 `发票号码.pdf` 命名。
- Excel 中的发票号码可自动匹配同名 PDF。
- 缺失同名 PDF 时，对应发票导入失败。
