# AGENT 全局知识库

## 1. 项目简介

本工作空间用于支持产品经理在 AI IDE 中完成高频工作，包括但不限于：

- PRD 撰写与迭代
- 字段清单整理与口径统一
- 前端 Demo 需求描述与验收
- 业务分析、流程推演与影响范围分析
- 汇报材料、交接文档与对外输出整理

本仓库默认将 `D:\Codexsdbb\src` 视为可运行 Demo 区域，将其余目录视为产品工作台。产品文档与 AI 协作内容应优先沉淀在约定目录内，避免信息分散。

## 2. 工作方式约定

### 2.1 AI 协作原则

- 所有正式结论优先沉淀到 `prds/`、`outputs/` 或 `docs/`
- 所有推演过程、草稿和中间思考优先放到 `drafts/` 或 `analysis/`
- 所有可复用提示词、规则和工作流优先沉淀到 `.agent/` 与 `prompts/`
- 当需求存在争议时，先补齐背景、目标、边界，再输出方案
- 当结论将影响研发排期或业务口径时，必须记录假设、风险和待确认项

### 2.2 默认产出标准

- 文档使用中文撰写，专业术语保留 English
- 关键结论尽量结构化，避免纯长段落
- 所有 PRD 至少包含目标、范围、流程、字段、异常场景、验收标准
- 所有字段清单至少包含字段名、类型、说明、是否必填、来源、校验规则
- 所有 Demo 需求至少包含页面目标、核心交互、状态变化、边界情况

## 4. 最佳实践

### 4.1 PRD 撰写

- 先写“为什么做”，再写“做什么”，最后写“怎么验收”
- 范围描述要区分“本期范围 / 非本期范围 / 依赖项”
- 页面需求要落到用户动作、系统响应、状态反馈三个层面
- 字段说明要同步写清展示规则、编辑规则和校验规则

### 4.2 字段清单整理

- 优先统一命名口径，再补字段级说明
- 对同名不同义字段加上业务前缀或上下文说明
- 对枚举值必须说明取值、含义、来源和默认值
- 对计算字段必须说明计算公式和更新时间点

### 4.3 业务分析

- 先识别目标指标和关键角色，再分析流程与约束
- 影响范围分析至少覆盖：前台、后台、接口、数据、运营、客服
- 流程推演至少覆盖正常路径、异常路径、回退路径
- 所有分析结论都要标注“事实 / 推测 / 待确认”

### 4.4 AI 使用

- 固定输出格式可以沉淀到 `prompts/` 或 `.agent/workflows/`
- 需要长期复用的判断规则优先写入 `.agent/rules/`
- 输出前要求 AI 自检：逻辑冲突、字段缺失、边界遗漏、验收不清
- 对重要交付物保留人工 review 记录，避免完全依赖一次生成

## 5. 经验教训

### 5.1 常见问题

- 只写页面说明，不写状态变化，导致研发实现口径不一致
- 只写主流程，不写异常场景，导致测试补洞成本高
- 字段命名随场景变化，后期口径难统一
- 输出分散在聊天记录里，后续检索成本高

### 5.2 规避建议

- 所有正式文档必须落盘，不依赖会话历史
- 所有 PRD 在评审前必须过一次字段完整性检查
- 所有 Demo 需求在交付前补齐交互、提示文案和边界状态
- 对容易反复出现的问题，沉淀为规则、模板或工作流
